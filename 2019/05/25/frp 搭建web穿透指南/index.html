类似的软件还有很多，比如ngrok,花生壳，向日葵等，由于他们是商业软件，受很多限制

总结

  1. 这次我们知道怎么搭建一个FRP服务，包括服务端和客户端的配置
  2. 参考链接 https://lfire.github.io/2017/11/15/deploy-the-frp-server/
  3. 80 端口共用，在一般的使用场景中，绝大多数的 WEB 服务是基于 80 端口提供服务的，因此，我们的 frp 不大可能在某一服务器上独占 80 端口资源，因此，这里启用了 28088 端口的兼听，通过 nginx 反向代理将某一域名的 80 端口请求转发到 28088。
  4. 泛域名解析，再结合 frp 的 subdomain 配置，实现多人共用 frp 服务器资源。
